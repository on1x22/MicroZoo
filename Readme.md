# Система по управлению деятельностью зоопарка (Microzoo)

Система предназначена для организации деятельности и контроля работы зоопарка, основанная на микросервисной архитектуре.

## Основные возможности:
* Планирование и мониторинг задач, выполяемых персоналом зоопарка
* Управление перечнем животных зоопарка

## Стек технологий:
* ASP.NET Core Web Api
* Docker
* PostgreSQL
* Entity Framework Core
* Api Gateway
* Unit Tests

## Микросервисы

Microzoo включает в себя следующие микросервисы:  
1. Api Gateway
2. Managers
3. Zookeepers
4. Persons
5. Animals

Общая схема взаимодействия представлена ниже  
![Scheme](https://github.com/on1x22/MicroZoo/blob/master/Scheme.png)

### 1. Api Gateway

Предназначен для предоставления единой точки входа в Microzoo для пользователя 
(работника зоопарка или менеджера) на основе библиотеки [Ocelot](https://ocelot.readthedocs.io/en/latest/). 
Подключение выполняется по протоколу https (порт 6001). В процессе своей работы Api осуществляет 
взаимодействие c микросервисами Managers и Zookeepers по протоколу http.

### 2. Managers Api

Позволяет управлять деятельностью зоопарка со стороны управляющего персонала.
Api предоставляет следующие возможности:

* Управление составом персонала зоопарка (как менеджеров, так и рядовых работников)
* Управление составом животных зоопарка
* Планирование и контроль выполнения задач работников
* Управление специализациями работников зоопарка

Доступ к микросервису предоставляется по порту 7004 по протоколу http. Для своей работы сервис 
получает информацию от Zookeepers, Persons и Animals по протоколу http.

### 3. Zookeepers Api

Микросервис предоставляет обычным работникам зоопарка информацию, необходимую для выполнения
своих повседневных задач:

* Просмотр, добавление и редактирование выполняемых работ
* Просмотр списка животных, за которых ответственен работник
* Просмотр и редактирование специализации (видов животных) работника зоопарка

Хранение информации о работах осуществляется в базе данных ZookeepersApi в таблице Jobs, о 
специализациях сотрудников - в таблице Specialities, к которым выполняет подключение сервис. 
Доступ к Zookeepers Api предоставляется по порту 7001 по протоколу http. Для своей работы 
сервис получает информацию от сервисов Persons и Animals по протоколу http.

### 4. Persons Api

Выполняет задачу по управлению общим списком и дополнительной информацией о персонале зоопарка 
(как менеджеров, так и обычных работников). Для этого микросервис выполняет подключение к 
БД PersonApi таблице PersonApi.  
Доступ к микросервису предоставляется по порту 7010 по протоколу http.

### 5. Animals Api

Микросервис необходим для просмотра, добавления, редактирования и удаления животных, 
находящихся в зоопарке, и информации о них. Информация о видах животных находится в 
БД AnimalsApi в таблице Animals, данные непосредственно о каждом животном - в таблице AnimalTypes.  
Доступ к микросервису предоставляется по порту 7020 по протоколу http.
